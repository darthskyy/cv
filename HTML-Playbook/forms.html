Practising using Forms

Generally forms are made of label and input elements and each label would serve one input (for this to occur the for attribute of the label and the id attribute of the input must match). For some inputs you can specify required to make sure it's filled in before it's submitted.

Text input

These are regular inputs which can take in any text data. You can limit the length of the text passed into the input using minlength and maxlength attributes. The type of the input is "text". You can also specify the placeholder and value attributes to put placeholders so that users know what to type in the input or default values

Example: text

Email input

This inherits all the syntax of a text input but with one added property that only allows for emails to be submitted. So if the input does not have an "@" it is not allow.

Example: email

Password input

Password is just a special type of input that masks the input.

Example: password

Checkbox input

This is for when users can select multiple values from a set of predefined values. You can give the attribute checked to checkboxes so that they are automatically selected from the beginning.

Example: checkbox



Radio input

These are similar to checkboxes on that you can only select one input for a set of radio inputs. to make sure that you can only select one input, you have to make sure that the name attribute for all the radios is set to the same value; this gives the mutual exclusivity. checked also applies here but for only one attribute in the set

Example: radio
